# 최종 상가+숙박 개수밀도 구현자료 0417

이 폴더는 최종 HTML을 다시 만들거나 검증할 때 필요한 입력 테이블, 보정 테이블, 최종 산출 테이블, 보조 파일, 재생성 스크립트를 모아둔 자료 폴더입니다.

## 최종 HTML 실행 파일

최종 HTML은 중복 저장하지 않고 아래 파일 하나만 유지했습니다.

```text
data\건물상가밀집도_10개구_0417\서울10개구_건물상가_50,000sqm_밀도지도.html
```

브라우저에서 열 파일은 이 HTML입니다. 이 폴더 안의 파일들은 HTML을 만들기 위해 사용한 자료와 보조 파일입니다.

## 최종 지표

- 격자 면적: 50,000㎡ = 5ha
- 최종 색상값: `상가숙박_개수_per_ha`
- 계산식: `(상가수 + 숙박시설수) / 5`
- 건물 규모 참고값: 건축면적, 연면적, 층수, 건축면적 x 층수 밀도
- 실제 건물 표시: `actual_buildings_10gu_0417.js`
- 0값 보정: 표제부 보정 후보 CSV를 활용해 SHP에서 면적/층수/이름이 0으로 들어온 건물을 복원
- 총층수: 각 테이블에서 `지상층수 + 지하층수`로 계산. SHP 원본은 한글 필드명이 깨지는 형식이라 `A29`를 `총층수` 필드로 사용

## 폴더 구성

```text
01_입력테이블
  ├─ 상가숙소_격자자료
  ├─ 건물원본_shp
  └─ 보정테이블
02_최종산출테이블
03_HTML보조파일
04_재생성스크립트
```

## 롯데월드타워 보정 메모

롯데월드타워는 원본 표제부에 존재하지만, SHP 실제건물 원본에서는 일부 건물 조각의 건축면적/연면적/층수/이름이 0 또는 공란으로 들어와 있었습니다. 이 때문에 기존 스크립트의 `밀도0 제외` 조건에서 빠졌고 최종 HTML의 실제건물 검색/호버 데이터에도 이름이 보이지 않았습니다.

현재는 `서울10개구_건물있지만_밀도0_표제부보정후보_건물목록.csv`를 이용해 해당 건물을 복원합니다. 보정된 실제 건물은 HTML에서 분홍색 점으로 표시되고, 호버에 건물명과 관리건축물대장 PK가 표시됩니다.
